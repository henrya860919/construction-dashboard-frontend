# 系統開發商與廠商關係與管理架構

本文件以**敘述方式**說明：你（系統開發商）與使用系統的廠商之間的關係、租戶與多租戶的差異、地端與集中部署的對應，以及平台方與廠商各自登入後如何進入單租／多租後台。供後續改程式時對照行為與動線。

---

## 一、角色與關係：誰是誰

你是**系統開發商**，也是這套 Construction Dashboard 的**平台營運方**。在系統裡你的身份是**平台管理員 (platform_admin)**，可以管理所有租戶（廠商）與其底下的專案、使用者。

使用你系統的**廠商**（例如廠商 A、廠商 B）在系統裡各對應一筆**租戶 (Tenant)**。一筆租戶就代表一家公司／組織，底下有該廠商的專案、使用者；各家廠商的資料依 `tenantId` 隔離，彼此看不到。

廠商內部會有兩種典型角色：**公司管理員**與**專案使用者**。公司管理員在系統裡是**租戶管理員 (tenant_admin)**，只管理「自己這家廠商」的專案與成員，使用的後台稱為**單租後台**（路徑 `/admin`）。專案使用者是**專案層使用者 (project_user)**，只會看到被指派的專案，進專案工作區做日常作業，不能進後台管理。

角色對應可簡記如下：

| 實際身份 | 系統中的對應 | 說明 |
|----------|--------------|------|
| 你（系統開發商） | platform_admin | 平台營運方，可管理所有租戶與資料 |
| 廠商 A / 廠商 B | 租戶 A、租戶 B (Tenant) | 一家廠商 = 一筆 Tenant，底下有專案與使用者 |
| 廠商的公司管理員 | tenant_admin | 只管理本廠商，進單租後台 |
| 廠商的專案使用者 | project_user | 只看被指派專案，進工作區，不進後台 |

## 二、租戶、多租戶，以及地端與集中部署

**租戶 (Tenant)** 在系統裡就是「一個客戶／一家公司」。一筆 Tenant 資料代表一家廠商，底下有該廠商的專案與使用者，資料用 `tenantId` 與其他租戶隔離。

**多租戶 (Multi-tenant)** 指的是：**同一套系統、同一套部署**同時服務**多個租戶**（多家廠商）。大家共用同一份程式與同一資料庫，靠 `tenantId` 區分誰的資料，由你（平台方）統一維運。相對地，**單租戶 (Single-tenant)** 則是「一套系統只服務一個租戶」；可以是多租戶系統裡只開一個租戶，也可以是**地端部署**——系統架在廠商那邊，那台機器只給該廠商用。

**地端部署**表示系統主機、資料庫架在**廠商自己的機房或辦公室**（on-premises）。每一家廠商一套獨立主機、獨立資料庫，所以那套實例通常就是**單一租戶**：只服務該廠商，沒有「多個租戶共用」；要服務第二家廠商就要在第二家再架一套，該實例上也**不會有多租後台**。**集中部署**則是系統架在**你（系統開發商）的機房或雲端**，多家廠商透過網路連到同一套系統登入；這就是**多租戶**，同一套系統上有多筆 Tenant，你用多租後台管理所有租戶。

本專案採用**多租戶 + 集中部署**：同一套程式、同一資料庫，多筆 Tenant 代表多家廠商；若未來改為地端部署給單一廠商，可只開一個 Tenant，該實例即單租戶使用。

**單租後台**（只管理「一個租戶」的專案、成員、設定）在概念上與**地端部署**最貼近：地端時那套實例上只會有「單租管理」；集中部署時則同時存在**多租後台**（給你管所有租戶）與**單租後台**（給各家廠商管理員管自己租戶）。單租後台是「管理範圍／介面」，地端是「架在哪裡」。

```
  地端部署（架在廠商那邊）                    集中部署（架在你這邊／雲端）
  廠商 A 機房一套、廠商 B 機房再一套          你這邊一套，廠商 A/B/C… 都連這套
  → 每套單一租戶，沒有多租後台                → 多租戶，依 tenantId 隔離，你用多租後台
```

## 三、平台與租戶的權限邊界

你（platform_admin）在**多租後台**（`/platform-admin`）可以管理所有租戶、全部專案、全部系統成員。各家廠商（租戶）的資料在底層依 `tenantId` 隔離；廠商管理員 (tenant_admin) 登入後只能看到、只能管理**自己這家租戶**的專案與成員，使用的介面是**單租後台**（`/admin`），看不到其他廠商。

也就是說：同一套系統裡有「平台層」與「租戶層」。平台層只有你看得到、管得到（多租後台）；租戶層則是每家廠商只看得到自己（單租後台，僅 tenant_admin 可進）。

```
  平台層：多租後台（租戶列表、全部專案、全部使用者）← 你 = platform_admin
      │
      │ 依 tenantId 隔離
      ▼
  租戶層：廠商 A 單租後台（專案、成員、設定）  │  廠商 B 單租後台（專案、成員、設定）
        僅 tenant_admin 可進                  │  僅 tenant_admin 可進
```

## 四、登入後動線

**平台管理員 (platform_admin) 登入後**，直接進入**多租後台**（`/platform-admin`）。側欄即顯示租戶管理、專案總覽、使用者總覽；不需再選「單租／多租」，也不顯示頂欄的「後台」按鈕（平台方只使用多租後台）。

**廠商管理員 (tenant_admin)** 登入後直接進**專案列表**（僅本租戶專案）。頂欄會出現**「後台」按鈕**，點選後可進**單租後台**（`/admin`）管理本廠商的專案、成員、公司設定，或從專案列表選專案進工作區。**專案使用者 (project_user)** 登入後也是進專案列表（僅被指派的專案），再選專案進工作區；沒有後台按鈕。

流程可簡化為：

```
  platform_admin 登入
        │
        ▼
  直接進入 /platform-admin
  側欄：租戶管理、專案總覽、使用者總覽
  （無頂欄「後台」按鈕）

  tenant_admin 登入
        │
        ▼
  專案列表 → 頂欄有「後台」→ 單租後台 /admin
```

## 五、管理功能與登入後畫面對照

| 角色 | 登入後預設／第一畫面 | 頂欄「後台」按鈕 | 多租後台（側欄） | 單租後台 | 專案內工作區 |
|------|----------------------|------------------|------------------|----------|--------------|
| platform_admin（你） | 直接進入多租後台（租戶管理、專案總覽、使用者總覽） | ✗ 不顯示 | ✓ 登入即在此 | ✗ 不開放 | ✓ 可進有權的專案 |
| tenant_admin（廠商管理員） | 專案列表（僅本租戶專案） | ✓ 顯示，可進單租後台 | ✗ 不可進 | ✓ 專案管理、成員管理、公司設定 | ✓ 本租戶專案 |
| project_user（專案使用者） | 專案列表（僅被指派專案） | ✗ 不顯示 | ✗ 不可進 | ✗ 不可進 | ✓ 被指派專案 |

---

## 六、流程摘要（供改程式對照）

1. **平台方 (platform_admin)**：登入 → 直接進 `/platform-admin`（多租後台），側欄為租戶管理、專案總覽、使用者總覽；頂欄無「後台」按鈕。於多租後台建立租戶、為各租戶建立第一個 tenant_admin 帳號。
2. **廠商管理員 (tenant_admin)**：登入 → 專案列表（本租戶）→ 頂欄「後台」→ 單租後台管理專案與成員，或選專案進工作區。
3. **廠商一般成員 (project_user)**：登入 → 專案列表（僅被指派專案）→ 選專案進工作區，無後台入口。
